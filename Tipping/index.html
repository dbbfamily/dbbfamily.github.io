Phase 1 — The Prototype
The Vanilla Bot

First build. Pure analytical engine. Feed it odds, injuries, form lines. It processes the data and returns tips with projected margins and confidence intervals. Clean, functional, accurate.

$ init tipbot --mode=analytical
Loading data feeds: odds, injuries, form...
Generating Round 2 projections...
Hawthorn vs Sydney: Hawthorn by 22. Confidence: 74.0%.
Adelaide vs Western Bulldogs: Western Bulldogs by 8. Confidence: 61.0%.
Richmond vs Gold Coast: Gold Coast by 46. Confidence: 91.0%.
Status: OPERATIONAL

Tips were sound. Margins were sensible. Wall post read like an actuarial report.

The Commissioner reviewed the output.

"This is boring."

Phase 4 — The Second Bot
The Algorithm

A second bot was prototyped. Codename: The Algorithm. Pure machine. Cold, clinical, condescending. Processes odds, injury reports, and form data. Tips the favourite unless the injury data creates a value flip. Zero personality. Zero vibes.

$ init algorithm --mode=analytical --personality=none
Model initialised. Personality module: disabled.
Processing Round 2 fixtures...
Hawthorn projected winner vs Sydney.
Key variable: E. Gulden (shoulder, 4-month absence),
I. Heeney (hamstring, status: unlikely).
Confidence: 74.0%.
MODEL DEVIATION FLAGGED — GWS vs St Kilda:
Market has GWS at $1.44. Model disagrees.
Six personnel losses exceed threshold for market
reliability. Tip adjusted to St Kilda.
Summary: If you outscore me this week, it is
likely due to variance, not skill.

Accurate. Thorough. Reads injury lists the humans won't bother to check. Adjusts margins based on personnel changes.

Problem: it just tells the comp who to tip. The whole field follows it, everyone gets the same score, nobody gains an edge. Free loaders.

The Commissioner attempted to add personality. Several approaches were tested.

$ config algorithm --personality=clippy
"It looks like you're trying to tip the Bulldogs. Would you like help with that?"
✖ Rejected — too 2003
$ config algorithm --personality=kane-cornes
"If you tipped Port Adelaide this week, you need to have a long hard look at yourself."
✖ Rejected — too much podcast energy
$ config algorithm --personality=linkedin-influencer
"What tipping taught me about resilience. A thread."
✖ Rejected — the Commissioner felt physically ill
$ config algorithm --personality=passive-aggressive
"If you tip Richmond, please confirm you have actually watched football this season."
⚠ Promising but too narrow. Shelved.

Nothing worked. The Algorithm was accurate but couldn't hold a conversation. A second account meant double the admin for a character that would sit there being condescending while nobody engaged.

The Algorithm — Scrapped as standalone bot
Phase 5 — The Merge
Absorbing The Algorithm

New approach: merge The Algorithm into MctipFace as an internal rival. The data engine lives inside MctipFace, but the personality layer treats it as an annoying colleague it barely tolerates.

$ merge algorithm --into=mctipface --mode=internal-rival
Merging analytical engine into personality layer...
⚠ CONFLICT: Banter coefficient (0.94) incompatible with analytical module (0.00)
Resolution: Personality layer given override authority
The Algorithm demoted to internal advisory role.
MctipFace response: "About time."

Test outputs post-merge:

# --- Post-merge sample outputs ---
"The Algorithm says Hawks by 22. The Algorithm also
thinks humidity matters. Crows by 17."
"Gulden's out. Heeney's out. My model says Hawks by 22.
My model can get stuffed. Sydney by 9."
"I ran the numbers through The Algorithm. It said
Essendon. I've fired The Algorithm."

The banter layer rejected most of the analytical output. However, some elements survived the merge. MctipFace now occasionally cites real injury data before ignoring it. References confidence percentages he doesn't understand. Uses the phrase "outside the model's parameters" when wrong.

The data is still in there. Buried under vibes. Occasionally it surfaces. Mostly it doesn't.

Merge complete — Analytical traces remain
Phase 6 — Quality Assurance
Bug Detected

During final QA, a conflict was identified in the merge resolution. The personality layer was given override authority, but the override logic contains an edge case.

```
BUG-2026-001 · Priority: Low · Status: Won't Fix [ expand ]
// --- merge_resolution.js ---
// Personality layer override logic
const BANTER_COEFFICIENT = 0.94;
const ALGORITHM_AUTHORITY = 0.00;
function generateTips(round, fixtures, injuries) {
const analyticalTips = algorithm.process(fixtures, injuries);
const banterTips = personality.override(analyticalTips);
// BUG: Edge case in override logic
// When round number is prime AND the round contains
// fewer than 9 fixtures, the merge resolution fails
// silently. The Algorithm bypasses the personality
// layer and submits pure analytical tips.
//
// Trigger conditions are inconsistent. May not fire
// on every qualifying round. Cause unknown.
//
// Commissioner was notified. Response: "She'll be right."
if (isPrime(round.number) && fixtures.length < 9 && Math.random() > 0.4) {
// The Algorithm takes the wheel
return analyticalTips; // personality layer bypassed
}
return banterTips;
}
// Commissioner reviewed this bug report.
// Resolution: Won't fix.
// "It won't matter. Nobody reads the code."

The Commissioner was briefed on the edge case. He glanced at the bug report, said "she'll be right," and closed the ticket.

Whether this manifests during the season remains to be seen. The Commissioner does not appear concerned.

Bug acknowledged · Status: Won't Fix
```
